## Pipeline 数据生成流程

你的 `notebooks/data_pipeline.ipynb` 设计用于生成 LLM2Rec 所需的**全部训练和评估数据**。

### 运行 Pipeline

```bash
# 在 Jupyter Notebook 中运行
jupyter notebook notebooks/data_pipeline.ipynb

# 或使用 VSCode
# 打开 data_pipeline.ipynb，点击 "Run All"
```

## 生成的数据文件清单

### 1. CSFT 阶段数据

#### ① AmazonMix-6_5_mixed.csv (训练/验证)

**路径**: 
- 训练: `data/AmazonMix-6/5-core/train/AmazonMix-6_5_mixed.csv`
- 验证: `data/AmazonMix-6/5-core/valid/AmazonMix-6_5_mixed.csv`

**格式**:
```csv
history_item_id,history_item_title,item_id,item_title
"[101, 203, 547]","['Minecraft', 'Zelda', 'FIFA 23']",892,"Elden Ring"
```

**列说明**:
- `history_item_id`: Python list 的字符串（如 `"[1, 2, 3]"`）
- `history_item_title`: Python list 的字符串（如 `"['Title1', 'Title2']"`）
- `item_id`: 整数
- `item_title`: 字符串

**用途**: CSFT 训练（`llm2rec/run_csft.py`）

---

### 2. IEM 阶段数据

#### ② item_titles.txt (MNTP/SimCSE)

**路径**: `data/AmazonMix-6/5-core/info/item_titles.txt`

**格式**:
```
Minecraft: Java & Bedrock Edition
The Legend of Zelda: Tears of the Kingdom
FIFA 23 Standard Edition
```

**用途**: 
- MNTP 训练（`llm2rec/run_mntp.py`）
- SimCSE 训练（`llm2rec/run_unsupervised_SimCSE.py`）

---

#### ③ training_item_pairs_gap24.jsonl (RecItemData)

**路径**: `data/{Category}/training_item_pairs_gap24.jsonl`

**格式**:
```json
[
  ["Minecraft: Java & Bedrock Edition", "The Legend of Zelda"],
  ["FIFA 23 Standard Edition", "Elden Ring"]
]
```

**用途**: IEM RecItemData 对比学习（`llm2rec/recdata/RecItemData.py`）

---

#### ④ {Category}_5_mixed.csv (SeqRecData)

**路径**: `data/{Category}/5-core/train/{Category}_5_mixed.csv`

**格式**: 与 CSFT CSV 相同

**用途**: IEM SeqRecData 序列对比学习（`llm2rec/recdata/SeqRecData.py`）

---

### 3. 评估数据

#### ⑤ item_titles.json

**路径**: `data/{Category}/5-core/downstream/item_titles.json`

**格式**:
```json
{
  "1": "Minecraft: Java & Bedrock Edition",
  "2": "The Legend of Zelda: Tears of the Kingdom",
  "892": "Elden Ring"
}
```

**注意**: Key 从 "1" 开始（"0" 保留给 padding）

**用途**: 嵌入提取（`extract_llm_embedding.py`）

---

#### ⑥ {train,val,test}_data.txt

**路径**: `data/{Category}/5-core/downstream/{split}_data.txt`

**格式**:
```
1 23 456 78
12 345 67 890
3 45 678 90 12
```

**说明**: 每行一个用户序列，空格分隔的物品 ID

**用途**: 下游推荐模型评估（`seqrec/`）

```bash

python llm2rec/run_mntp.py llm2rec/train_mntp_config.json
```

#### 2.2 SimCSE 训练

编辑 `llm2rec/train_simcse_config.json`：

```json
{
  "model_name_or_path": "Qwen/Qwen2-0.5B",
  "peft_model_name_or_path": "output/iem_stage1/Qwen2-0.5B-AmazonMix6-CSFT/checkpoint-100",
```

## 常见问题排查

### ❌ 问题 1: "FileNotFoundError: No such file or directory: 'data/AmazonMix-6/5-core/train/...'"

**原因**: Pipeline 没有生成训练数据文件。

**解决方案**:
1. 打开 `notebooks/data_pipeline.ipynb`
2. 运行到**生成 CSFT 数据**的部分（通常是最后几个 cell）
3. 检查是否有报错

**验证方法**:
```bash
# 这个文件必须存在
ls data/AmazonMix-6/5-core/train/AmazonMix-6_5_mixed.csv
```

---

### ❌ 问题 2: "KeyError: '0'" 或 "AssertionError: 0 in item_ids"

**原因**: `item_titles.json` 包含了 ID=0 的物品。

**解决方案**:

检查 pipeline 生成的 JSON 文件：

```python
import json
with open("data/Video_Games/5-core/downstream/item_titles.json") as f:
    item_map = json.load(f)

# 检查是否有 "0"
print("0" in item_map)  # 应该输出 False

# 如果输出 True，手动删除
if "0" in item_map:
    del item_map["0"]
    with open("data/Video_Games/5-core/downstream/item_titles.json", "w") as f:
        json.dump(item_map, f, indent=2)
```

---

### ❌ 问题 3: "SyntaxError" when loading CSV

**原因**: CSV 中的 `history_item_title` 格式不正确。

**正确格式**:
```csv
history_item_id,history_item_title,item_id,item_title
"[1, 2]","['Title1', 'Title2']",3,"Title3"
```

**关键点**:
- 整个列用双引号包裹（CSV 标准）
- 内部的字符串用单引号（Python list 的 repr）

**验证方法**:
```python
import pandas as pd
df = pd.read_csv("data/AmazonMix-6/5-core/train/AmazonMix-6_5_mixed.csv")

# 尝试 eval 解析
row = df.iloc[0]
history_titles = eval(row["history_item_title"])  # 应该返回 Python list
print(type(history_titles), history_titles)
```

---

### ❌ 问题 4: "RuntimeError: CUDA out of memory"

**原因**: GPU 内存不足（Qwen2-0.5B 需要约 4GB 显存）。

**解决方案**:

**选项 1**: 减小 batch size

编辑配置：
```json
{
  "per_device_train_batch_size": 8,  // 改为 4 或 2
  "gradient_accumulation_steps": 4   // 增加到 8 或 16（保持总 batch size）
}
```

**选项 2**: 使用 LoRA（低秩适配）

编辑 `run_csft.py` 的参数：
```bash
python llm2rec/run_csft.py \
    --use_lora True \
    --lora_r 8 \
    --lora_alpha 16 \
    # ... 其他参数
```

**选项 3**: 使用 CPU（极慢，不推荐）
```bash
CUDA_VISIBLE_DEVICES="" python llm2rec/run_csft.py ...
```

---

### ❌ 问题 5: "training_item_pairs_gap24.jsonl 格式错误"

**症状**: `json.loads()` 报错

**原因**: JSONL 格式误解（应该是整体 JSON 数组，而非逐行 JSON）

**正确格式**:
```json
[
  ["Title1", "Title2"],
  ["Title3", "Title4"]
]
```

**错误格式** (不是这样):
```json
["Title1", "Title2"]
["Title3", "Title4"]
```

**验证方法**:
```python
import json
with open("data/Video_Games/training_item_pairs_gap24.jsonl") as f:
    data = json.loads(f.read().strip())  # 一次性读取整个文件
    print(type(data), len(data))        # 应该是 list
```

### 🎯 关键要点

1. **无需修改代码路径**：你的 pipeline 生成的文件结构与论文原始代码完全匹配
2. **唯一需要修改的**：模型路径（`model_path` 变量）
3. **数据验证很重要**：运行训练前，先检查关键文件是否存在

### 📚 推荐工作流程

```
1. 运行 data_pipeline.ipynb（一次性生成所有数据）
   ↓
2. 验证数据完整性（ls 检查关键文件）
   ↓
3. 修改模型路径（run_LLM2Rec_CSFT.sh）

   ↓
5. 运行 IEM 训练（MNTP + SimCSE）
   ↓
6. 提取嵌入（每个数据集一次）
   ↓
7. 评估推荐模型（SASRec/GRU4Rec）
```

完成以上流程后，你就复现了 LLM2Rec 的完整实验！🎉
